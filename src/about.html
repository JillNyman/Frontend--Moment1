Arbetsprocessen

Node.js har installerats, som gör det möjligt att exekvera kod JavaScript-kod utanför webbläsaren. Tillsammans med Node.js installerades Node Package Manager, npm, som behövs för hantering av JavaScript-paket och bibiliotek. Vid installationen av npm skapas en mapp med namnet node-modules, och den exkluderas från projektet genom att ange den i .gitignore-filen.

För att automatisera en del av processen installerades Parcel, som bland annat upptäcker och paketerar moduler automatiskt, har stöd för flera filtyper och man behöver därför inte konfigurera relationen mellan dem manuellt. Paketet Sharp installerades för att kunna visa bilder i andra format än i original (t ex webp).

För att källkodsfilerna (HTML, CSS, JS) ska ta mindre plats vid en publicering (det vill säga komprimeras) sker via Parcel en optimering av koden i publiceringsfilerna. Det kan till exempel vara att CSS-filerns slås ihop och på andra sätt görs så liten som möjligt.

Projektet laddas upp via Git och filerna optimeras automatiskt vid en commit och push i en process som brukar kallas Continuous Integration/Continuous Deployment.

Bakåtkompabiliteten automatiserades för att inte behöva koda manuellt för äldre webbläsare. Detta genom att i package.json-filen lägga till rubriken browserslist och där ange vilka versioner bakåt som webbplatsen också ska anpassas för.

Kurs: Frontendbaserad webbutveckling, 7,5 hp
